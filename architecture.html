medium.en model, CPU int8, beam_size=5, VAD filter enabled

Mode Router
Explicit mode switching: "switch to chat/command mode". Default: command mode.
orchestrator.py — run_agent()
Routes to one of two modes
Command Mode
App detection, auto-focus, shortcut injection, short-circuit checks, then RAG + LLM tool calling
command_router.py + app_index.py
Conversation Mode
Chat with LLM, 20-turn history, temp=0.7, no tool calling
conversation.py
Command: App Detection + Auto-Focus
Longest n-gram match against app index. If detected: focus window (or launch via GNOME Search + wait up to 5s). Pre-fetch and inject keyboard shortcuts into prompt.
command_router.py — prepare_command_context() | app_index.py
Short-Circuit Check (20+ patterns)
Common commands (focus, mute, screenshot, brightness, battery, etc.) execute directly without LLM. Response time <100ms.
command_router.py — try_short_circuit()
RAG Namespace Selection
Cosine similarity on SentenceTransformer embeddings (all-MiniLM-L6-v2). Top K=2 namespaces. Forced routing: window verbs → window ns, app name → input ns.
app_index.py — retrieve_relevant_namespaces(), build_filtered_tool_schema()
LLM Tool Calling - Gemma 4 E4B (Q4_K_M)
llama-server on localhost:8081, Vulkan GPU (Intel Arc), temp=0.0, max 5 chaining steps. Truncation detection with auto-retry at 600 tokens.
llm_chain.py — run_chain()
Tool Execution
Facade tools route to MCP server (anthony-mcp). Standalone tools call system commands directly. Results feed back to LLM for chaining.
tools/facades.py, tools/standalone.py — MCP calls via mcp_client
TTS Response - Piper Neural Voice
Strip markdown, pad short text with fillers, synthesize WAV, play via aplay
voice_io.py — speak()

External Systems

llama-server

  • Gemma 4 E4B (Q4_K_M, 8B params)
  • OpenAI-compatible API on :8081
  • Vulkan GPU (Intel Arc), 99 layers
  • 4096 token context, ~6-7 tok/s

anthony-mcp

  • MCP server over stdio
  • Window, input, settings, media
  • Screenshots, search, notifications
  • System control (battery, brightness, power)
  • Runs as GNOME Shell extension

Voice Models

  • Piper TTS (en_US-lessac-medium ONNX)
  • Faster-Whisper (medium.en, int8 CPU)
  • Silero VAD (PyTorch)
  • All loaded at startup

GNOME Desktop

  • Mutter window manager (D-Bus)
  • gsettings (dark mode, a11y)
  • AT-SPI2 (dialog detection)
  • Gio.AppInfo (app indexing)

System Tools

  • upower (battery)
  • brightnessctl (brightness)
  • systemctl/loginctl (power/lock)
  • flatpak + dnf (app management)

Safety Mechanisms

🛡
Dialog detection
AT-SPI/dogtail detects save dialogs on close. Reads button labels dynamically, prompts user by voice, activates choice by keyboard.
🔌
Automation health checks
Auto-enables GNOME extension if disabled. Auto-recovers on "disabled" or "not responding" errors during tool execution.
⚠
Power action confirmation
Suspend, restart, shutdown, and logout require explicit voice confirmation before execution.
📦
Install confirmation
App install/uninstall searches first, presents options, waits for user selection and confirmation.
🔄
Tool chain limit
Max 5 chaining steps per command prevents infinite LLM loops. Speaks last result if limit reached.
🚨
Error recovery
try/except/finally around command processing. LLM errors, MCP failures, and tool exceptions don't crash the orchestrator.
